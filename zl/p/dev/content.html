Development

DV1. Information schema

In theory this is important part of web-development (we provide information and this is our product). But in real life nobody cares. Do it quick:

steps to do

  • rewriting content in plain text
  • suggesting placeholders for missing content
    lorem ipsum, temporary images and media
  • html markup
    using browser defaults for html semantics
    HTML semantics

goals

  • all content is available in one place
  • content organized with groups/sections
  • missing content is marked

    missing data
    • in forms
      labels, error messages, success feedback, ...
    • in products
      unique url, special prices, testimonials, ...
    • in lists
      filters, sorting functions, search fields, ...
    • in project
      404, page titles, ...
  • content is print-friendly

DV2. Content layout

headers, footers, collumns...

steps to do

  • positioning on screen
    all components should have right place and time on the screen
  • positioning inside components
    • gallery items and controls
    • product cards
    • articles, lists and all others
  • making everything adaptive and responsive
  • making everything independent from content
    set min- and max- values

goals

  • html templates are ready for backend team
  • task to visual design is ready
    ┬Фmake this ugly templates to be beautiful, please┬╗

DV3. Navigation

steps to do

  • defining user routes and URL design
  • creating project controls
    page transitions, links, navigation bars
  • creating component controls
    tabs, pagers, modals, collapse and others
  • covering navigation with tests
    Mocha
    Jasmine

goals

  • clickable prototype
    as complete project without design and backend
  • all transitions are defined
    create a task for motion design
  • avoid broken links and buttons

DV4. Forms

steps to do

  • searching for solutions to avoid the forms
  • correct sizes and types
  • making large forms to be user-friendly
    • dividing into groups
    • adding suggestions, autofills and default values
  • making form elements to be client-friendly
    here clients = browsers and devices
  • creating validation and error reports
  • covering forms with tests

goal

  • user-friendly forms

DV5. Create communication with content and backend

steps to do

  • creating usefull api
    data-binding, data-model, views, preloads
  • defining updates and version control

Frontend design

If previous steps are reached, the design step becomes very predictable. Precize timing allows to stop programming in a fixed time, moving undeveloped functionality to the next release.

FD1. Create communication with design

steps to do

  • to discuss the design task, using prototype
  • for design-oriented projects:
    • compare prototype with current design
    • define missing parts
  • create checklist for frontend design

FD2. Animations

steps to do

  • defining animations
    • for layout transitions
    • for interactive elements
    • for content (banners, decorations)
  • creating animations
    • using SVG
    • using CSS
    • GIF, Adobe Flash, js-libraries, ...
    SVG animation
  • check for correct time and timing functions
  • check for browser performance

FD3. Create design system

steps to do

  • creating atoms (basic elements)
    titles, buttons, images, labels, links ...
  • creating components
    headers, articles, product cards, galleries ...
  • creating templates for ┬Фhtml-email┬╗
  • checking and confirming design system

FD4. Creating designed prototype

steps to do

  • joining everyhing together
    • clickable prototype
    • designed components
    • current content
  • making content print-friendly
  • making content internet-friendly
    for search engines, social and advertise networks
  • visual testing
    • check all devices
    • check all states
    • imitate errors (broken JS, no CSS, ...)
  • creating areas for technical debt
    with custom.css/custom.js places

goals

  • content is usefull
  • content looks good
  • bussiness task is reached
